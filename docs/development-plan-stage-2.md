# FlowLedger 开发流程规划（Stage 2）

最后更新：2026-05-23

Stage 1 总结见 [development-plan.md](development-plan.md)。

## 1. 阶段目标

Stage
1 把核心业务跑通了，但 UX 是「功能验证型」——首页信息密度过大、导航靠 sidebar 卡片堆叠、缺统计、没有为日常长期使用打磨。

Stage 2 的判断标准是：**自己能愿意每天用，且部署到 Vercel + Turso 上线长期跑**。

具体三件事：

1. **手机优先的 IA 重做**：底栏 Tab Bar 取代 sidebar 卡片，二级管理页统一归类。
2. **首页与核心视图重新设计**：按「每天 10 秒 / 每周 1 分钟 / 每月 5 分钟」三种节奏分层。
3. **统计分析**：5 个核心视图（月度对比、分类排行、月度趋势、净资产、信用卡总览），轻量实现（Tailwind
   div 画图，不引图表库）。

完成后才部署。

## 2. 设计原则

延续 Stage 1 已经定下的：

- 单代码 + mobile-first CSS
- Tailwind v4 + shadcn/ui (base-nova)，不引新 UI 组件库
- 不重做交易领域模型、不重做表单组件本身
- 数据模型尽量不动；只动 UI / 路由 / 信息架构

新增的：

- **手机底栏导航是主入口**。所有"管理"性质的页面都归到第 4 个 tab。
- **首页只放每天用的东西**。低频维护性入口（账户管理、模板维护、汇率、信用卡设置）都不在首页直接出现。
- **统计图不上图表库**。用 `<div className="h-{n}">` + 颜色 + 数字搞定。够用、加载快、可控。

## 3. 个人使用清单（驱动设计的事实）

判断"放首页还是二级"的依据：

| 频率     | 动作                         | 放在哪                            |
| -------- | ---------------------------- | --------------------------------- |
| **每天** | 记一笔（快捷或临时）         | 首页主区 + 底栏「记一笔」浮动按钮 |
| 每天     | 看本月还剩多少钱 / 本月支出  | 首页 hero 区数字                  |
| **每周** | 看上周/本周花在哪里          | 统计 tab：分类排行                |
| 每周     | 确认信用卡待还               | 管理 tab：信用卡                  |
| **每月** | 校准钱包 / 现金余额          | 管理 tab：账户 → 详情 → 校准      |
| 每月     | 月度总览（vs 上月 / 三月均） | 统计 tab：月度对比                |
| 每月     | 处理待退款 / 待确认周期项    | 首页：待办区                      |
| 偶发     | 新建账户 / 改汇率 / 改模板   | 管理 tab                          |

## 4. 里程碑

### S2.1 导航系统：底栏 Tab Bar + 路由分层

目标：所有页面归到 4 个顶级入口，手机上拇指可达。

任务：

- 新建 `src/components/ui/bottom-tab-bar.tsx`，4 个 tab：
  - 首页（/）
  - 记一笔（/transactions/new 或 modal — 待定）
  - 统计（/stats — S2.4 才有内容，先占位）
  - 管理（/manage — S2.3 才有内容，先占位）
- 在 `src/app/layout.tsx` 渲染底栏；`/login` 不显示
- 底栏视觉：fixed bottom + safe-area-inset，64px 高（含 safe area），active tab 主题色
- 桌面 `@media (min-width: 820px)` 隐藏底栏，恢复或保留现有顶部导航
- 把现有 sidebar 卡片里的「管理周期项 / 退款 / 信用卡 / 分期 / 快捷模板 / 退出登录」入口迁出，准备并入 /manage（S2.3）
- 现有 /accounts / /transactions / /recurring 等 URL 不变，只是入口移到底栏

验收：

- 手机视口（375px）底栏始终可见、不被键盘遮挡
- 切换 tab 路由切换、active 视觉正确
- 桌面视口底栏隐藏

### S2.2 首页（Dashboard）重设计

目标：打开首页 10 秒内能完成 80% 的日常动作。

任务：

- Hero 区：本月支出 / 本月结余 / 折算净资产 三大数字，跟随系统暗色模式
- 月度进度感：本月已过 N 天的隐性提示，比如"已过 18/30 天，已花 ¥85k"
- 快捷模板网格上移到主区（不再藏在 sidebar）
- 待办区合并：N 个待确认周期项 + N 笔退款未到账 +（新增）N 笔本月信用卡待还，按紧急度排序
验收：

- 手机 iPhone 15（393×852）一屏能看到：hero + 快捷网格至少 4 个
- 滚动一屏看完所有：hero + 快捷 + 待办 + 最近交易
- 桌面看起来不空旷（响应式布局调整）

### S2.3 「管理」二级页面整合

目标：所有低频维护性页面归到一个 tab，视觉风格统一。

任务：

- 新增 /manage 入口页：列表式聚合，每条点进对应子页
  - 账户管理（→ /accounts）
  - 快捷模板（→ /templates）
  - 周期项目（→ /recurring）
  - 信用卡（→ /credit-cards）— 含新增 CRUD
  - 退款追踪（→ /refunds）
  - 分期计划（→ /installments）
  - 汇率（→ 新建 /manage/rates）
  - 数据导出（→ 新建 /manage/export）
  - 退出登录（按钮，调用 logoutAction）
- **补 S1 遗留**：
  - 信用卡 CRUD UI（新建信用卡需要联动创建 account + credit_cards 两条记录）
  - 汇率编辑页 /manage/rates：列出所有汇率对，可改
  - 数据导出 /manage/export：导出 transactions / accounts CSV（按时间范围）
- 每个子页头部的「返回首页」改为「返回管理」（如果是从 /manage 进来的）

验收：

- /manage 一个页面能跳到所有维护功能
- 信用卡能从 UI 完整创建（不需要手改数据库）
- 汇率能从 UI 改（不需要 SQL）
- CSV 导出能下载，含本月 / 全部 两个选项

### S2.4 统计分析

目标：5 个核心视图回答 5 个核心问题。所有图都用 Tailwind div 画。

任务：

- /stats 路由，第 3 个底栏 tab
- **视图 1：月度对比** — 本月 vs 上月 vs 三月均，支出 / 收入 / 结余三组数字，差异百分比
- **视图 2：分类排行** — 本月支出按 category 排序，每行：分类名 + 金额 + 横向进度条（宽度 = % of
  total）+ 笔数。可切月份。
- **视图 3：月度趋势（12 个月）** — 每月一根竖条，高度 = 当月支出 / max(12个月)，鼠标 hover
  / 点击显示数字。
- **视图 4：净资产** — 当前 JPY +
  CNY + 折算总，对比 30 天前的差额。每个账户一行进度条占总净资产的比例。
- **视图 5：信用卡总览** — 每张卡本期消费 / 待还 / 扣款日，已逾期红色突出。
- 顶部时间范围切换：本月 / 近 3 月 / 近 12 月 / 全部
- 全部用 server component 直接查库，不引图表库

验收：

- 5 个视图都能加载在 1 秒内（本地 SQLite）
- 月度趋势在 365px 视口下不挤
- 时间范围切换通过 URL searchParams，可分享

### S2.5 部署上线

目标：M6.5 真正部署，FlowLedger 进入个人长期使用。

任务：

## 5. 跨阶段约定

继承 Stage 1 的 [code-guidelines.md](code-guidelines.md) 与 [ui-direction.md](ui-direction.md)：

- 新写代码继续 mobile-first / shadcn / useActionState / MoneyInput 那套
- domain 函数有单元测试（vitest）
- 所有删除带 AlertDialog 确认
- 删 globals.css 的旧自定义 class 时，先确认没有遗留引用

新增约定：

- **底栏 tab 之间的页面不互相强耦合**：从「统计」点出去能回到「统计」，不要互相 redirect 乱跳
- **统计页只读**：不在统计页提供编辑入口（一致性 vs 上下文之间，选一致性）
- **「管理」页面不在 dashboard 首页上重复出现入口**：除非该子页有 actionable 状态（如"3 笔退款待到账"）

## 6. 推迟到 Stage 3 的东西

避免 Stage 2 范围蔓延。以下明确不进 Stage 2：

- 跨币种转账 + FX 手续费记录
- 标签系统（schema 字段 + 输入 UI + 标签筛选）
- 朋友代付 / AA / 借出借入专门状态
- 月末结账向导
- 自动汇率同步（仍是手动维护）
- 多用户 / OAuth / Auth.js 迁移
- 信用卡账单确认状态（用户已确认过这期账单）
- 投资资产（股票 / 基金 / 加密货币）

如果用了一段时间发现真的需要，加进 Stage 3。

## 7. 风险

- **范围蔓延**：「重新设计」很容易变成「重写」。每个 milestone 完成前先回看任务清单，超出的进 Stage
  3 backlog。
- **底栏 + 安全区联调**：Vercel 默认域名 vs 添加到主屏后 PWA 模式的 safe-area 行为不一致。S2.5 真机测试时重点验。
- **统计性能**：现在交易量小，server
  component 直接 sum 没问题。等数据量上千笔可能要加索引。先不优化。
- **数据迁移**：Stage
  2 不动 schema 是目标，但若发现 IA 改造需要新字段（例如本月预算），评估必要性再加。

## 8. 推荐下一步

S2.1 开始：先做底栏，把现有 sidebar 卡片入口迁出。这一步不涉及业务变化，验证底栏视觉 + 路由切换正确即可。
