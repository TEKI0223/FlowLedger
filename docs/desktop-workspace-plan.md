# FlowLedger PC 桌面工作台计划

最后更新：2026-05-26

## 1. 背景

当前 FlowLedger 的主要体验是手机优先 UI。现有首页、交易、账户、管理等页面都围绕手机端日常记账设计，桌面宽度下虽然可以使用，但并不是完整的 PC 管理工作区。

这次探索的目标不是替换手机端，而是确认未来是否需要一套独立的 PC 桌面端：用户在桌面端进入系统后，能够停留在桌面 UI 内完成查看、筛选、维护、编辑和处理待办，而不是频繁跳回手机优先页面。

## 2. 当前已完成

已完成一个 `/desktop` v0 原型，定位是“桌面总览入口”，不是完整桌面端替代版。

完成内容：

- 新增 `/desktop` 桌面总览页。
- 新增桌面 shell：
  - 左侧固定导航。
  - 顶部搜索。
  - 快捷记账入口。
  - 新建交易入口。
  - 主题切换。
  - 退出登录。
- `/desktop` 首屏展示：
  - 折算净资产、本月收入、本月支出、本月结余、待处理数量。
  - 账户资产表。
  - 最近交易表。
  - 待处理事项。
  - 基础资料入口。
- `/desktop` 前缀下隐藏原 `BottomTabBar`，避免桌面工作台内出现手机端底部导航。

验证记录：

```bash
npm run lint
npm run build
```

以上命令已通过。浏览器验证 `/desktop` 可打开，且原 `/manage` 页面仍保持旧导航和旧页面结构。

## 3. 当前局限

当前 `/desktop` v0 仍然主要是“更强首页/总览入口”，还不是完整 PC 工作区。

主要局限：

- 桌面左侧导航中的交易、账户、信用卡、周期项目、退款、分期、统计、分类、模板、支付方式、汇率等入口仍会跳到原有手机优先路由。
- 顶部“快捷记账”“新建交易”仍会进入 `/entry`，也就是现有手机优先录入页。
- 账户详情、交易详情、编辑、新建、还款、确认周期项目、退款到账、分期操作等核心流程还没有
  `/desktop/*` 版本。
- 因此，目前不能保证桌面端全流程不接触手机 UI。

结论：当前成果应视为桌面端信息架构和视觉密度的 v0 探索，不应被理解为完整 PC 端改造完成。

## 4. 暂缓原因

完整桌面端并不是技术上特别困难，真正的不确定性在产品需求。

如果现在立即补齐所有 `/desktop/*` 页面，会产生两个风险：

- PC 端真实使用方式还不够明确，容易做出一批“看起来完整但不一定顺手”的页面。
- 现有手机端流程已经可用，过早大规模复制/改造页面会增加维护成本。

因此，当前先暂停完整桌面路由开发，只保留 `/desktop` v0 原型作为后续需求讨论和设计验证的基础。

## 5. 后续方向

后续如果继续推进，建议采用独立 `/desktop/*` 路由方案，而不是把现有手机端路由改成同路由响应式重构。

推荐顺序：

1. 交易闭环
   - `/desktop/transactions`
   - `/desktop/transactions/[id]`
   - `/desktop/entry`
   - 目标：在桌面端完成交易查看、筛选、详情查看和新建记录。
2. 账户闭环
   - `/desktop/accounts`
   - `/desktop/accounts/[id]`
   - 账户充值、消费、校准、编辑入口。
   - 目标：在桌面端完成账户资产查看和余额维护。
3. 基础资料管理
   - 分类、快捷模板、支付方式、汇率。
   - 目标：用桌面表格和紧凑编辑页替代手机列表式维护体验。
4. 计划类功能
   - 信用卡、周期项目、退款、分期。
   - 目标：桌面端可以处理账单、确认周期项目、追踪退款和查看分期进度。
5. 统计和分析
   - 在桌面端提供更高信息密度的分析视图。
   - 目标：使用更宽的布局展示趋势、排行、资产结构和信用卡概览。

## 6. 设计原则

- 手机端保持稳定，不为了桌面端改坏现有日常记账体验。
- 桌面端使用独立 `/desktop/*` 路由，避免桌面改造污染手机端。
- 复用现有 `data.ts`、`service.ts`、server actions 和领域模型。
- 桌面 UI 重点放在：
  - 表格。
  - 筛选。
  - 搜索。
  - 详情侧栏。
  - 快捷操作。
  - 更高信息密度。
- 不急于一次性补齐所有模块。每次推进都应该完成一个可用闭环，而不是只做入口。

## 7. 当前状态标记

状态：**v0 原型已完成，完整桌面路由暂缓。**

后续恢复开发前，建议先明确：

- PC 端最高频的 3 个操作是什么。
- 是否需要桌面端编辑/删除/批量维护能力。
- 是否希望 PC 端访问旧手机路由时自动提示或跳转到 `/desktop/*`。
- 桌面端更像“数据表格工作台”，还是更像“财务分析控制台”。
